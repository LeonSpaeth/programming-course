Arbeiten mit Variablen: Operatoren

Wir haben gesehen, wie man mittels Variablen auf Speicher zugreifen kann. Wenn wir allerdings nichts mit den Werten in unserem Speicher tun können, als sie zu lesen und zu schreiben, können wir keine besonders aufregenden Dinge damit tun. Was uns fehlt sind Operatoren, um Variablen und Konstanten auf verschiedene Weise miteinander zu verknüpfen. Zum Beispiel wollen wir gelegentlich zwei Werte miteinander addieren. Also brauchen wir den Operator +. Welche Operatoren könnten uns noch interessieren, um vernünftig rechnen können?

Zuvor haben wir die Nullstellen einer quadratischen Funktion berechnet. Wollen wir das nur ein einziges Mal ausführen, mag das in Ordnung sein. Ein Kommentar zu Kommentaren

In den meisten Programmiersprachen kann man Kommentare schreiben. Sie werden beim übersetzen/interpretieren des Codes ignoriert. Daher müssen sie sich nicht an die Syntax der Sprache halten. Sie dienen dazu, in natürlicher Sprache den Code zu kommentieren. Das kann zum Beispiel helfen eine komplizierte Stelle im Code verständlicher zu machen. Allerdings werden sie häufig missbraucht, um unverständlichen Code zu rechtfertigen. Außerdem sind sie weniger präzise als der Code (immerhin ist der Sinn von Code eine möglichst präzise Beschreibung). Somit können sie eher irreführen, als aufklären. Insbesondere wird es dann gefährlich, wenn der Code verändert wird (was sehr häufig passiert) und der Kommentar der alte bleibt. Das Motto sollte daher lauten: anstatt schwer verständlichen Code zu kommentieren, sollte er so geschrieben werden, dass man ihn ohne Kommentare versteht.

Öffne die Datei comments.js. Dort siehst du Kommentare an zahlreichen Stellen im Code. Dein Ziel ist, alle Kommentare loszuwerden. Dabei solltest du nicht einfach alle Kommentare entfernen, denn so wie der Code geschrieben ist, wird es ohne Kommentare kaum möglich sein, ihn zu verstehen. Stattdessen solltest du den Code verändern, damit er selbsterklärend wird. Die Funktion sollte allerdings dieselbe bleiben.
